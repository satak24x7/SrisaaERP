# User Stories

Reference these IDs (US-01 … US-22) when writing tests, UI screens, or acceptance criteria. Each module's FR table links back to these.

| ID | Story | Acceptance signals |
|---|---|---|
| US-01 | As Priya, I want every statutory document, certification and DSC to have a validity date, owner and reminder so that nothing expires unnoticed before a critical bid. | Expiry dashboard; 90/60/30-day alerts; no bid disqualified for expired docs |
| US-02 | As Rajesh, I want to see the full pipeline across Managed and Standard Tenders with probability-weighted value so that I can prioritise pursuit effort. | Single pipeline view with filters; weighted forecast updates live |
| US-03 | As Rajesh, I want Cost-of-Sale figures on every opportunity so that go/no-go decisions reflect true economics, not just contract value. | Opportunity card shows spent + committed + projected cost vs contract value |
| US-04 | As Meera, I want to generate a draft bid pack (eligibility, certifications, CVs, past performance) in under an hour so that I have time to focus on the technical solution. | Draft pack generated in < 60 min from org + employee master |
| US-05 | As Meera, I want a compliance matrix that is auto-seeded from the RFP requirements checklist so that nothing is missed. | Compliance matrix template generated per RFP; PM signs off before submission |
| US-06 | As Arjun, I want the Kanban board to reflect the current milestone and auto-surface blocked tasks so that my standups are data-driven. | Kanban filtered per milestone; blocked swim-lane visible; standup view < 3 min |
| US-07 | As Arjun, I want to see Estimated vs Committed vs Actual cost by line-item at any time so that I can intervene before overruns happen. | Budget-vs-Actual view refreshes daily; variance alerts at 80% and 100% |
| US-08 | As Kiran, I want to log task status and expense bills on my phone in under 60 seconds so that updating the system doesn't steal field time. | Mobile status update ≤ 3 taps; expense submission ≤ 60 sec with photo attach |
| US-09 | As Suresh, I want a rolling 12-month cash-flow forecast that combines milestone invoicing schedules and retention releases so that I can plan funding. | Rolling forecast view; exportable; updates on invoice / receipt events |
| US-10 | As Suresh, I want to track PBGs and retention across all projects with auto-reminders before release or expiry so that capital is not locked unnecessarily. | PBG / retention register with validity alerts and release workflow |
| US-11 | As Deepa, I want a one-page portfolio dashboard of opportunities, projects, revenue and risks so that I can steer the business without asking ten people. | CEO dashboard available on web and mobile; refreshes daily |
| US-12 | As Rajesh, I want to manage upstream (Managed Tender) engagements with stakeholder maps and DPR versions so that long-cycle pursuits don't lose continuity when staff change. | Managed Tender record preserves all stages, stakeholders, DPR versions |
| US-13 | As Vikram (BU Head), I want a single dashboard of all opportunities and projects in my Business Unit so that I can spot trouble early and plan interventions. | BU Portfolio dashboard with funnel, project RAG, expense and procurement pipeline |
| US-14 | As Vikram, I want approvals for go/no-go, budgets, expense sheets and MRs above threshold to come to me in one queue so that I can act quickly without digging through email. | Unified approval inbox with filters, bulk approve, mobile action |
| US-15 | As Kiran, I want to submit a weekly expense sheet on mobile with photo-captured bills so that I don't lose receipts and get reimbursed faster. | Mobile expense sheet submission ≤ 2 min; photo → OCR assist; status tracking |
| US-16 | As Arjun, I want to approve or return expense sheets and MRs for my project in bulk, with budget availability visible at decision time, so that approvals don't become a bottleneck. | PM approval queue with budget meter; bulk approve; return-for-clarification |
| US-17 | As Meera, I want to book pre-project expenses (travel, demos, consultant fees) against the specific Opportunity so that we know the true Cost of Sale before the bid is submitted. | Pre-Project Expense Sheet tagged to Opportunity and Cost of Sale budget |
| US-18 | As Kiran, I want to raise a Material Request from the site with item codes, quantities and required-by date, and see it progress through to issue, so that the material actually arrives when I need it. | MR form (mobile); status visible from request → PO → GRN → issue |
| US-19 | As Neha (Admin), I want to consolidate approved MRs across projects into a single indent and run an RFQ so that volume buying reduces cost and effort. | Indent consolidation screen; RFQ capture; vendor selection with comparison |
| US-20 | As Ravi (Stores), I want to issue material against an approved MR and capture digital acknowledgement from the receiver so that accountability is clear. | Issue Slip with e-acknowledgement; stock ledger updates on issue |
| US-21 | As Suresh, I want every approved expense and PO to reduce available budget on the linked project or opportunity in real time so that we cannot over-commit without explicit approval. | Commitment accounting visible on project budget view; over-commit requires override |
| US-22 | As Neha, I want a vendor scorecard with OTIF, quality rejects and payment cycle so that I can make repeat-order decisions on evidence, not relationship alone. | Vendor scorecard report with rolling 12-month metrics |
